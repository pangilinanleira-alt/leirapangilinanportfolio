  • Lesson 1: JavaScript

  • Overview and History
    JavaScript was developed by Brendan Eich at Netscape. It was originally named Mocha, then LiveScript, before becoming JavaScript. It is considered the most popular and widely used scripting language on the internet and is supported by all major browsers. The name "JavaScript" is a trademark of the Oracle Corporation.

    Core Technical Characteristics
    It is a client-side, interpreted, and object-oriented language. Client-side Processing**: Code is processed directly on the user's computer rather than the server. The program is passed as source code and converted into machine code. It is case-sensitive and supports structured programming syntax similar to C. While it uses semicolons at the end of statements, they are not strictly required. The basic components of the language are objects, methods, and properties.

    Capabilities and Use Cases
    JavaScript adds functionality to websites, allowing for real-time updates, personalized feeds, and video controls. Event Handling**: It can react to user actions like mouse clicks and page pre-loading. Data Validation**: It is used to validate user input in forms, saving processing time by handling it before sending it to a server. JS can create and manage cookies to save or retrieve visitor information. When combined with CSS, it creates DHTML, which allows for visual effects like message scrollers or falling snow.

  • Lesson 2: JavaScript Guidelines

  • Placement of Scripts

    JavaScript can be placed in different sections of an HTML file to control when the code executes.

    HEAD Section: Scripts placed here are "pre-loaded," meaning they execute before a user triggers any events. This section is ideal for function calls.

    BODY Section: Scripts in the body are executed as the page loads.

    Multiple Placements: A developer can use as many script tags as needed in both the `head` and `body` sections.

    External JavaScript

    Using external `.js` files is efficient for controlling multiple HTML files without rewriting code.

    Implementation: External files are linked using the `src` attribute within a script tag.

    Comments

    Comments are used for explanations or debugging and are ignored by the browser: Single-line: Indicated by two backslashes (`//`). Multiple-line/Block: Start with `/*` and end with `*/`. JavaScript Coding Guidelines

    To prevent errors and improve readability, the document lists three primary guidelines:

    Case Sensitivity**: Unlike HTML, JavaScript is case-sensitive. For example, the variable `myHome` is different from `MyHome`.

    White Space: JavaScript generally ignores white space within a line of code. Using spaces between expressions is recommended for readability.

    Breaking Up Code Lines**: A backslash (`\`) can be used to break a line of code. However, code should never be broken in the middle of a command (e.g., `document.\write` is invalid).

  • Lesson 3: Characteristics of JavaScript

  • JavaScript is an interpreted language, meaning it runs through a web browser instead of being compiled. It is also a scripting language used to perform simple tasks and control parts of a webpage rather than creating the whole page. JavaScript also has client-side scripting, where JavaScript runs on the user’s computer through the browser, unlike server-side scripts that run on a server. Another important feature is Object-Oriented Programming (OOP), where elements in a webpage are treated as objects that have properties and methods which can be controlled or changed using JavaScript.
